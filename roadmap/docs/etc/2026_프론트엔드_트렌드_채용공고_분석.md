## 1. Must-Have 기본 역량

> 어떤 회사든 반드시 요구하는 기본 역량

| 역량 | 상세 | 시장 수요 |
|------|------|-----------|
| **TypeScript** | 문법이 아닌 깊은 이해 (제네릭, 유틸리티 타입, 타입 추론) | 필수 |
| **React** | 컴포넌트 아키텍처, 훅, 상태 관리 | 50~60만 관련 채용 (글로벌) |
| **상태 관리** | 프레임워크 문법보다 "설계" 능력 (언제 어떤 상태를 어디에 둘지) | 필수 |
| **성능 기본** | Core Web Vitals (LCP, CLS, INP), 코드 스플리팅, 번들 최적화 | 필수 |
| **테스트** | 단위 테스트 작성, 테스트 피라미드 이해 | 필수 |
| **접근성(A11y)** | WCAG 2.1 AA, 시맨틱 마크업, 키보드 네비게이션 | 필수 (선택→필수로 전환됨) |
| **Git** | 전문적인 협업 워크플로우, PR 리뷰 | 필수 |
| **REST API** | API 설계/소비, HTTP 프로토콜 이해 | 필수 |

## 2. Trending 트렌디한 역량 (2026 핫한 스킬)

> 적극적으로 찾고 있고 프리미엄 연봉을 주는 역량

### 프레임워크 & 아키텍처

| 역량 | 왜 트렌디한가 | 애슬러 적용 가능성 |
|------|--------------|-------------------|
| **Next.js** | React 채용공고의 71%가 요구. 풀스택 React의 기본 | SDUI와 유사한 서버 주도 패턴 → 개념 비교 학습 가능 |
| **React Server Components (RSC)** | 2026년 기업 도입 본격화. 서버/클라이언트 분리 | SDUI가 이미 서버 주도 UI → 철학적 공통점 분석 가치 |
| **Full-stack 마인드셋** | 서버 이해 필수. 프론트만 아는 개발자는 한계 | AI DA 서버 개발 경험 + BE 이해 → 이미 방향성 맞음 |
| **Monorepo 실전** | 대규모 프로젝트 관리 핵심 | Yarn Berry workspaces 이미 사용 중 → 최적화 여지 |

### 성능 & 최적화

| 역량 | 왜 트렌디한가 | 애슬러 적용 가능성 |
|------|--------------|-------------------|
| **React Compiler** | v1.0 출시(2025.10). 수동 메모이제이션 대체 | 직접 적용 검토 가능 |
| **Edge Computing** | CDN 엣지에서 로직 실행. 응답 시간 획기적 단축 | 개념 학습 + SDUI 캐싱에 활용 가능성 |
| **번들 최적화** | CWV가 매출/SEO에 직결. 성능 = 비즈니스 | webpack 설정 최적화, 코드 스플리팅 강화 |
| **이미지 최적화** | WebP/AVIF, responsive, lazy loading | 패션 커머스는 이미지가 핵심 → 높은 ROI |

### 테스트 & 품질

| 역량 | 왜 트렌디한가 | 애슬러 적용 가능성 |
|------|--------------|-------------------|
| **Playwright** | 2026년 E2E 테스트 표준. Cypress 대체 | 페이지 어드민에 이미 계획 → 확장 가능 |
| **Visual Regression** | 스크린샷 기반 UI 변경 감지. 디자인 시스템 안정성 | ui-system 컴포넌트 검증에 적합 |
| **자동화된 접근성 테스트** | CI/CD에서 WCAG 자동 검증 | 웹 빌드에 추가 가능 |
| **성능 테스트** | CI에서 Core Web Vitals 자동 측정 | Lighthouse CI 도입 |

### AI 통합 (가장 큰 차별화 요소)

| 역량 | 왜 트렌디한가 | 애슬러 적용 가능성 |
|------|--------------|-------------------|
| **LLM 코딩 워크플로우** | AI를 활용한 개발 생산성 향상이 채용 핵심 평가 기준 | Claude Code 이미 활용 중 → 고도화 |
| **AI 코드 리뷰** | AI가 PR 리뷰, 보안/성능 이슈 탐지 | SuperClaude 에이전트 활용 |
| **프롬프트 엔지니어링** | 프론트엔드 맥락에서의 효과적 LLM 상호작용 | skill/rule 시스템으로 이미 실천 중 |

## 3. Emerging 미래 역량 (다음 물결)

> 아직 메인스트림은 아니지만 빠르게 성장하는 영역

| 역량 | 성장 방향 | 투자 시기 |
|------|-----------|-----------|
| **Concurrent React** | useTransition, useOptimistic, Suspense 활용 | 지금 학습 시작 |
| **Streaming HTML** | Progressive Hydration, Streaming SSR | 개념 이해 단계 |
| **Agentic AI 워크플로우** | 자율적 AI 에이전트 활용 개발 | 실험 단계 |
| **RAG (Retrieval-Augmented Generation)** | LLM + 외부 데이터 연결 | AI DA와 연관 |
| **WebAssembly (WASM)** | 성능 크리티컬 프론트엔드 코드 | 장기 투자 |
| **Web Components** | 프레임워크 독립적 컴포넌트 | 개념 이해 |
| **Service Workers & PWA** | 오프라인 기능, 캐싱 전략 | 웹앱 강화 시 |

## 4. 한국 IT 시장 특수 요구사항

### 채용 트렌드 변화

| 변화 | 상세 |
|------|------|
| **채용 규모 축소, 기준 상향** | 양보다 질. 더 깊은 기술 이해 요구 |
| **경험 > 이론** | 프로젝트 경험이 가장 중요한 평가 기준 |
| **Full-stack 기대** | 프론트만 하는 개발자보다 서버도 이해하는 개발자 선호 |
| **AI 활용 능력 필수화** | AI 도구를 다룰 줄 아는 개발자에 대한 수요 급증 |
| **데이터 기반 사고** | 기능 구현이 아닌 지표 변화를 만들어내는 능력 |

### 회사별 특징

**토스 (Toss)**
- 빠른 배포 + 극한 성능 추구 문화
- 채용: 3년 이상 경력, 1개월 채용 프로세스
- 핵심: "빠르게 만들고, 빠르게 최적화하라"

**카카오 (Kakao) / 카카오페이**
- 클라우드 인프라, 서비스 스케일링, 분산 시스템 강조
- 상시 채용 + 3년 이상 경력 요구
- 결제 플로우 신뢰성, 보안 중시

**네이버 (NAVER)**
- 엔터프라이즈급 프론트엔드, 대규모 트래픽 처리
- Full-stack 사고, 대규모 최적화 경험

**라인 (LINE)**
- 클라이언트 플랫폼(웹, 모바일)
- 크로스 플랫폼 경험, 실시간 커뮤니케이션 UX

**당근 (Karrot)**
- 로컬 커머스, 커뮤니티 기반 플랫폼
- React Native 경험, 모바일 최적화

## 5. 애슬러에서 바로 경험할 수 있는 트렌디 역량

> 새로운 기술을 배우기 위해 사이드 프로젝트를 할 필요 없이, 애슬러 앱에서 바로 경험할 수 있는 것들

| 트렌디 역량 | 애슬러에서 어떻게 경험하는가 |
|------------|--------------------------|
| **Monorepo 실전** | Yarn Berry 4.1.0 + 7개 패키지 운영 중 → 최적화/빌드 효율화 |
| **SDUI 아키텍처** | 서버 주도 UI 직접 설계/운영 → RSC와 철학적 유사성 어필 |
| **크로스 플랫폼** | iOS/Android/Web 단일 코드베이스 → React Native + Web 동시 지원 |
| **커머스 핵심 플로우** | 결제/쿠폰/장바구니/로그인 → 안정성/정합성 경험 |
| **성능 최적화** | RN New Architecture + Web CWV → 플랫폼별 성능 전략 |
| **AI 활용 개발** | Claude Code + SuperClaude → AI 협업 워크플로우 실전 |
| **서버 개발** | AI DA (AWS Lambda) → Full-cycle 경험 |
| **Playwright 테스트** | 페이지 어드민 UI 테스트 → E2E 자동화 경험 |
| **TypeScript 심화** | 브랜드 타입, 유니온 타입, Zod 활용 → 타입 안전성 강화 |
| **상태 관리 설계** | Zustand + React Query + SDUI → 복잡한 상태 아키텍처 경험 |

## 6. 요약: 액션플랜에 반영할 핵심

### 즉시 반영 (P0 트렌디)
1. **React Compiler** 학습 및 적용 검토
2. **Playwright E2E** 핵심 플로우 테스트 체계화
3. **Visual Regression Testing** 도입
4. **Core Web Vitals CI 통합**
5. **접근성 기본기** 학습

### 중기 반영 (P1)
6. **RSC 개념** → SDUI와 비교 분석
7. **Concurrent React** 패턴 활용
8. **번들 최적화** + 성능 예산

### 장기 반영 (P2~P3)
9. **Edge Computing** 개념 이해
10. **Agentic AI** 워크플로우 확장
